# TRELLIS.2 部署聊天记录

> 日期：2026-07-19
> 本文只整理用户的关键提示词，以及助手给出的关键判断和手动操作步骤。

## 1. 部署目标

### 用户

我想在 Windows 11 上部署 TRELLIS 2，最终效果是能本地启动一个网页，在网页里上传一张图片，就能生成一个 3D 模型，且能够下载这个 3D 模型到本地。你来一步步教我怎么做，我来手动部署。

电脑配置：

```text
Python 3.11.9
CUDA Toolkit 13.2
NVIDIA Driver 595.97
NVIDIA GeForce RTX 5090 D v2
24GB 显存
约 64GB 内存
```

TRELLIS.2 模型地址：

<https://huggingface.co/microsoft/TRELLIS.2-4B>

### 助手关键回复

微软官方 TRELLIS.2 当前只测试 Linux，并要求至少 24GB NVIDIA 显存。RTX 5090 的显存达到最低要求。最初建议使用 Windows 11 + WSL2 Ubuntu，因为官方原生 Windows 支持不完整。

Windows 中已经安装的 Python 3.11 和 CUDA Toolkit 13.2 不需要卸载。WSL 环境原计划使用独立的 Python 和 CUDA 依赖。

## 2. 检查 WSL

### 用户

执行：

```powershell
wsl --status
```

输出：

```text
未安装适用于 Linux 的 Windows 子系统。可通过运行 “wsl.exe --install” 进行安装。
```

### 助手关键回复

建议用管理员 PowerShell 安装：

```powershell
wsl --install -d Ubuntu-24.04
```

正常情况下首次启用 WSL2 后需要重启 Windows。

## 3. 网吧不能重启保留环境

### 用户

我正在网吧，重启电脑后安装的 Ubuntu-24.04 会消失，能不能不重启电脑？

### 助手关键回复

可以先尝试逐条执行：

```powershell
wsl --install --no-distribution
wsl --update
wsl --install -d Ubuntu-24.04 --web-download
```

其中：

- `wsl --install --no-distribution` 安装 WSL 基础组件，不安装 Linux 发行版。
- `wsl --update` 更新 WSL 运行组件和 Linux 内核，不是下载 Ubuntu。
- `wsl --install -d Ubuntu-24.04 --web-download` 才是安装 Ubuntu 24.04。

这些命令应逐条执行。任何一步明确要求重启时应停止。

## 4. WSL2 路线确认不可用

### 用户

执行后输出：

```text
已安装 适用于 Linux 的 Windows 子系统 2.7.10。
正在安装 Windows 可选组件: VirtualMachinePlatform
操作成功完成。
请求的操作成功。直到重新启动系统前更改将不会生效。
```

随后：

```text
wsl --update
已安装最新版本的适用于 Linux 的 Windows 子系统。

wsl --install -d Ubuntu-24.04 --web-download
请求的操作成功。直到重新启动系统前更改将不会生效。
```

### 助手关键回复

该电脑首次启用 `VirtualMachinePlatform`，必须重启后才能启动 WSL2，当前会话无法绕过。

如果网吧还原系统会在重启后撤销 Windows 功能，那么 WSL2 无法使用。继续反复运行安装命令不会生效。

## 12. 下次继续时的关键手动步骤

由于网吧重启后所有磁盘都会还原，下次需要从头重复本记录第 6 至第 10 节的原生 Windows 安装步骤。

环境重新安装并验证后：

1. 登录 Hugging Face，确认 DINOv3 审批已经通过。
2. 设置 F 盘缓存环境变量。
3. 下载 DINOv3。
4. 下载 TRELLIS.2-4B。
5. 启动 ComfyUI。
6. 加载图片转 3D 示例工作流。

离开网吧前，把生成的 GLB、输入图片、修改后的工作流和本聊天记录保存到外部存储；留在任何本地盘中的文件都会在重启后消失。
